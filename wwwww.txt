Absolutely! Let’s break down **how to handle meta tags per product and per category** in a Laravel 8 e-commerce project. I’ll provide **concepts, database structure, implementation, and sample code**.

---

## 1️⃣ Concept

Meta tags help SEO by providing search engines information about your page. The key meta tags are:

* **Title** (`<title>`): Page title, usually product or category name.
* **Description** (`<meta name="description">`): Short summary for search engines.
* **Keywords** (`<meta name="keywords">`): Optional, but sometimes used.
* **OG Tags** (`<meta property="og:title">`, etc.): For social sharing.

In e-commerce:

* **Per Product**: Each product can have a unique title, description, and optional keywords.
* **Per Category**: Each category can have a title, description, and keywords summarizing all products in it.

---

## 2️⃣ Database Design

You can either:

**Option A: Add meta columns to existing tables**

```sql
-- products table
ALTER TABLE products
ADD COLUMN meta_title VARCHAR(255) NULL,
ADD COLUMN meta_description TEXT NULL,
ADD COLUMN meta_keywords TEXT NULL;

-- categories table
ALTER TABLE categories
ADD COLUMN meta_title VARCHAR(255) NULL,
ADD COLUMN meta_description TEXT NULL,
ADD COLUMN meta_keywords TEXT NULL;
```

**Option B: Use a separate `seo_meta` table**

This is more flexible if you want meta tags for other pages too.

```sql
CREATE TABLE seo_meta (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    metaable_id BIGINT NOT NULL,
    metaable_type VARCHAR(255) NOT NULL,
    meta_title VARCHAR(255) NULL,
    meta_description TEXT NULL,
    meta_keywords TEXT NULL,
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL
);
```

* `metaable_type` = `App\Models\Product` or `App\Models\Category`
* `metaable_id` = the ID of the product or category

---

## 3️⃣ Models in Laravel

### Option A: Columns in tables

```php
// Product.php
class Product extends Model
{
    protected $fillable = [
        'name', 'price', 'description', 
        'meta_title', 'meta_description', 'meta_keywords'
    ];
}

// Category.php
class Category extends Model
{
    protected $fillable = [
        'name', 'slug', 'meta_title', 'meta_description', 'meta_keywords'
    ];
}
```

### Option B: Polymorphic relation

```php
// SeoMeta.php
class SeoMeta extends Model
{
    protected $fillable = ['meta_title','meta_description','meta_keywords'];

    public function metaable()
    {
        return $this->morphTo();
    }
}

// Product.php
public function seoMeta()
{
    return $this->morphOne(SeoMeta::class, 'metaable');
}
```

---

## 4️⃣ Controller Example

```php
// ProductController.php
public function show($slug)
{
    $product = Product::where('slug', $slug)->firstOrFail();

    // If using columns
    $meta_title = $product->meta_title ?? $product->name;
    $meta_description = $product->meta_description ?? substr($product->description,0,160);
    $meta_keywords = $product->meta_keywords ?? implode(',', $product->tags->pluck('name')->toArray());

    return view('products.show', compact('product', 'meta_title', 'meta_description', 'meta_keywords'));
}

// CategoryController.php
public function show($slug)
{
    $category = Category::where('slug', $slug)->firstOrFail();

    $meta_title = $category->meta_title ?? $category->name;
    $meta_description = $category->meta_description ?? 'Check out all products in '.$category->name;
    $meta_keywords = $category->meta_keywords ?? implode(',', $category->products->pluck('name')->toArray());

    return view('categories.show', compact('category', 'meta_title', 'meta_description', 'meta_keywords'));
}
```

## 5️⃣ Blade Template Example

```blade
<!-- resources/views/layouts/app.blade.php -->
<head>
    <title>{{ $meta_title ?? config('app.name') }}</title>
    <meta name="description" content="{{ $meta_description ?? '' }}">
    <meta name="keywords" content="{{ $meta_keywords ?? '' }}">
    
    <!-- Open Graph -->
    <meta property="og:title" content="{{ $meta_title ?? config('app.name') }}">
    <meta property="og:description" content="{{ $meta_description ?? '' }}">
    <meta property="og:url" content="{{ url()->current() }}">
    <meta property="og:type" content="website">
</head>
```

* Pass `$meta_title`, `$meta_description`, `$meta_keywords` from your controller to the view.
* Blade will automatically render them for SEO.

---

## 6️⃣ Optional: Default Meta Tags

If a product or category does not have meta tags, you can set **defaults in `AppServiceProvider`**:

```php
View::composer('*', function($view){
    $view->with('meta_title', config('app.name'));
    $view->with('meta_description', 'Default site description');
    $view->with('meta_keywords', 'shop, ecommerce, products');
});
```

1. Add meta columns or a polymorphic `seo_meta` table.
2. Fill meta tags for each product and category.
3. In controllers, use meta fields if present, else use defaults.
4. Render in Blade with `<title>` and `<meta>` tags.
5. Optional: add OG tags for social sharing.

Ah! I see exactly what you mean — your **product page Blade view is a child of a layout** (e.g., `layouts.app`) in Laravel 8. In that case, you can **pass meta tags from the controller to the child Blade**, and have the layout render them dynamically. Here’s the full approach:

---

## 1️⃣ Blade Layout (Parent)

```blade
<!-- resources/views/layouts/app.blade.php -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <title>@yield('meta_title', config('app.name'))</title>
    <meta name="description" content="@yield('meta_description', 'Default site description')">
    <meta name="keywords" content="@yield('meta_keywords', 'shop,ecommerce')">

    <!-- Open Graph -->
    <meta property="og:title" content="@yield('meta_title', config('app.name'))">
    <meta property="og:description" content="@yield('meta_description', 'Default site description')">
    <meta property="og:url" content="{{ url()->current() }}">
    <meta property="og:type" content="website">

    @stack('head') <!-- optional for extra tags -->
</head>
<body>
    @yield('content')
</body>
</html>
```

> Here, `@yield('meta_title')` allows child pages to inject their own meta tags.

---

## 2️⃣ Product Page (Child Blade)

```blade
<!-- resources/views/products/show.blade.php -->
@extends('layouts.app')

@section('meta_title', $meta_title)
@section('meta_description', $meta_description)
@section('meta_keywords', $meta_keywords)

@section('content')
    <h1>{{ $product->name }}</h1>
    <p>{{ $product->description }}</p>
    <p>Price: ${{ $product->price }}</p>
@endsection
```

* The `@section('meta_title', $meta_title)` syntax passes variables from the controller to the layout.
* Blade will render the parent `<head>` with these values.

---

## 3️⃣ Controller Example

```php
// ProductController.php
public function show($slug)
{
    $product = Product::where('slug', $slug)->firstOrFail();

    $meta_title = $product->meta_title ?? $product->name;
    $meta_description = $product->meta_description ?? substr($product->description, 0, 160);
    $meta_keywords = $product->meta_keywords ?? implode(',', $product->tags->pluck('name')->toArray());

    return view('products.show', compact('product', 'meta_title', 'meta_description', 'meta_keywords'));
}
```

* You **prepare meta variables in the controller**.
* The child Blade passes them to the parent layout using `@section`.

---

## 4️⃣ Optional: Default Meta Tags in Layout

If you forget to pass a meta tag, `@yield('meta_title', config('app.name'))` will fall back to a default.

---

### ✅ Advantages of this approach:

1. Works perfectly with **Blade child pages** (`@extends`).
2. Each page can have **unique meta tags**, including product, category, blog, etc.
3. Centralized in **layout**, so you don’t repeat `<head>` in every Blade.
